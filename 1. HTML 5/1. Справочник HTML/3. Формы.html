Далее идут примеры работы всех остальных атрибутов input'a

> name - значение: текст

- инпут с name='input1'

> value - значение: текст

- с помощью value можно устанавливать стандартное значение

> form - значение: id формы

- теперь можно получить доступ к этому элементу по id = 'id777'

> size - значение: число

- вот такая ширина при size='10'

> placeholder - значение: текст

- наведи на элемент курсор, и получишь подсказку!

атрибуты - Флаги:

> autocomplete - значение: on / off

> autofocus - значение: нет, просто autofocus

- элемент с autofocus
- автоматически получает автофокус при загрузке страницы. Автофокус получает самый нижний элемент с autofocus

> checked - значение: нет, просто checked

- чекбокс 1
- чекбокс 2 - включен по умолчанию
- чекбокс 3

> disabled - значение: нет, просто disabled

- чекбокс 1
- чекбокс 2 отключен (disabled)
- чекбокс 3

> readonly - значение: нет, просто readonly

- этот элемент только для чтения (редактировать не получится)

атрибуты - Для type == submit / image / file:

> src - значение: локальный адрес или URL

- к кнопке типа image можно приделать картинку, например такую. Напоминаю, это аналог submit. Кнопке заданы атрибуты src='ссылка на картинку', width и height - задающие ширину и высоту, и alt - с текстом картинки для поисковиков

> height - значение: пиксели

> witdh - значение: пиксели

> alt - значение: текст

> formaction - значение: локальный путь или адрес

- кнопка 1, если ее нажать то данные из формы отправятся на обработку скрипту index1.php
- кнопка 2, если ее нажать то данные из формы отправятся на обработку скрипту index2.php
*Примечание: это все кнопки одной и той же формы

> formenctype - значение: см. выше у атрибута Enctype

- кнопка 1, если ее нажать то при отправке данные будут закодированы методом "application/x-www-form-urlencoded"
- кнопка 2, если ее нажать то при отправке данные будут закодированы методом "multipart/form-data"
- кнопка 2, если ее нажать то при отправке будут данные закодированы методом "text/plain"
*Примечание: это все кнопки одной и той же формы

> formmethod - значение: get / post

- кнопка 1, если ее нажать то данные будут отправлены HTTP-методом POST
- кнопка 2, если ее нажать то данные будут отправлены HTTP-методом GET
*Примечание: это все кнопки одной и той же формы

> formtarget - значение: см. у атрибута Target

- кнопка 1, если ее нажать ответ от формы будет получен в окно _blank
- кнопка 2, если ее нажать ответ от формы будет получен в окно _self
- кнопка 3, если ее нажать ответ от формы будет получен в окно _parent
- кнопка 4, если ее нажать ответ от формы будет получен в окно _top
*Примечание: это все кнопки одной и той же формы

> accept - значение: audio/* / video/* / image/* / MIME_type


- атрибутом accept мы здесь уточнили, что сервер должен принять файл типа image/jpeg, и благодаря этому в открывшемся окне выбора файла можно будет выбрать только этот тип файла

атрибуты - Валидация:

> required - значение: нет, просто required

- у текстового поля установлен required - значит не получится отправить данные, пока в текстовом поле не будет валидной информации

> pattern - значение: RegExp

- значение в текстовом поле валидуется регулярным выражением. Также обязательно включить required. Должна быть 1 или более цифр. Не работает в IE<10 и safari

> formnovalidate - значение: нет, только formnovalidate

- у инпута типа email, в котором встроенная валидация, указан флаг formnovalidate, отключающий какую-бы то нибыло валидацию

атрибуты - Значение элемента:

> max - значение: число, дата-время

- тип input со значением max=100, min=0, value=20

> min - значение: число, дата-время

> maxlength - значение: число

- это инпут типа текст с МАХ длинной текста maxlength='5' символам

> step - значение: число

- тип input со значением max=100, min=0, value=20 и шагом step=10

> multiple - значение: нет, просто multiple

- тут указан флаг multiple, можно выбрать для загрузки несколько файлов одновременно

> list - значение: id от datalist

- попробуй вписать firefox, chrome, opera или safari, и сработает автокомплит, потому что это предопределенные опции из datalist
